## 🚀 Как запустить

Понадобится любой локальный сервер — двойной клик по `index.html` **не сработает**
(браузеры блокируют 3D-модули у файлов, открытых «с диска», адрес `file://`).

**Вариант 1 — терминал (самый простой):**

```bash
cd sites/astra-casino
npx serve .
```

Откройте адрес, который появится в терминале (обычно `http://localhost:3000`).

**Вариант 2 — VS Code / WebStorm:**
правый клик по `index.html` → «Open with Live Server» (в WebStorm — открыть через встроенный браузерный сервер).

**Вариант 3 — Python (если есть):**

```bash
cd sites/astra-casino
python3 -m http.server 8000
# открыть http://localhost:8000
```

> [!IMPORTANT]
> Нужен интернет: Three.js, GSAP, Lenis и шрифты подключаются через CDN.
> Видео, картинки и звук — **локальные**, лежат в `assets/`.

## ✏️ Как сделать сайт своим

Всё редактируется в обычном текстовом редакторе. Основной файл — `index.html`,
цвета и шрифты — в начале `css/style.css`. Идите по шагам.

### Шаг 1. Название бренда

Откройте `index.html` и через поиск (`Ctrl+F` / `Cmd+F`) найдите все `ASTRA`.
Название встречается здесь:

| Где | Что искать |
|---|---|
| Вкладка браузера | `<title>` в самом верху файла |
| Описание для поисковиков | `<meta name="description">` рядом с `<title>` |
| Шапка сайта | ссылка `class="brand"` в `<header>` |
| Первый экран | строка-надпись «ASTRA · Beyond the expected» в `id="hero"` |
| Блок «О проекте» | текст `class="about-lead"` в `id="about"` |
| Подвал | ещё одна ссылка `class="brand"` + подпись `class="footer-tag"` «Beyond the expected» |
| Пункт меню | ссылка «Мир Astra» в шапке и подвале |

Дескриптор бренда — **«Beyond the expected»** («Игра за пределами ожидаемого»).

### Шаг 2. Секции и тексты

Каждая секция в `index.html` подписана своим `id` — найдите нужную и поменяйте текст внутри:

| Секция | `id` в файле | Что внутри |
|---|---|---|
| Шапка | `id="siteHeader"` | Логотип, меню, кнопка звука |
| Первый экран | `id="hero"` | Видео, заголовок, подзаголовок, две кнопки |
| Каталог игр | `id="games"` | Фильтр-чипсы + шесть игровых блоков |
| Манифест | `id="manifesto"` | «Не громче. Глубже.» и три принципа |
| О проекте | `id="about"` | Описание концепта и короткие факты |
| Окно демо | `id="gameModal"` | Всплывающее видео игры (открывается по кнопке «Открыть демо») |

### Шаг 3. Шесть игр

Список игр собран **в одном месте** — файл `js/main.js`, массив `games` в самом начале.
Из него берутся фильтр по категориям и содержимое всплывающего окна:

```js
export const games = [
  { id: 'nova-reels',     title: 'Nova Reels',     category: 'slots',   poster: 'assets/img/nova-reels.webp',     video: 'assets/video/nova-reels' },
  { id: 'prism-roulette', title: 'Prism Roulette', category: 'live',    poster: 'assets/img/prism-roulette.webp', video: 'assets/video/prism-roulette' },
  // ... ещё четыре
];
```

Чтобы переделать игру под свою:

1. Положите свой постер `assets/img/<id>.webp` и ролик `assets/video/<id>.mp4` (и `.webm`, если есть) — **с теми же именами**, что в `id`.
2. Поменяйте `title` и `category` в массиве `games`, если нужно.
3. Обновите заголовок и описание в самом блоке игры в `index.html` (у каждого блока есть атрибут `data-game-block="<id>"`).

> [!TIP]
> Категории для фильтра (`slots`, `live`, `instant`, `table`) должны совпадать с
> названиями чипсов-фильтров в секции `id="games"`. Больше про сами видео и картинки —
> в [`assets/README.md`](assets/README.md).

### Шаг 4. Контакты и форма

Их здесь **нет специально**. ASTRA — витрина атмосферы, а не рабочее казино,
поэтому в шаблоне нет формы регистрации, депозитов и полей ввода данных.
Кнопки «Исследовать игры», «Смотреть шоурил» и «Открыть демо» просто листают к
играм или открывают демо-ролик — никакие данные никуда не отправляются.

Если делаете на основе шаблона реальный продукт — форму, вход и приём данных
подключайте **своим** проверенным бэкендом и с соблюдением закона (18+, лицензии,
ответственная игра). Это ответственность того, кто использует шаблон.

### Шаг 5. Цвета и шрифты (по желанию)

Вся палитра собрана в самом начале `css/style.css`, в блоке `:root` — поменяли
значение в одном месте, и оно поменялось по всему сайту:

```css
:root {
  --bg: #060813;               /* почти чёрный фон */
  --surface: #0d1020;          /* панели и карточки */
  --surface-elevated: #12162a; /* приподнятые блоки */
  --text: #f2f1f7;             /* основной текст */
  --text-muted: #a5a3b5;       /* приглушённый текст */
  --violet: #7b5cff;           /* фиолетовое свечение и глубина */
  --cyan: #4df3e5;             /* главный акцент: кнопки, ссылки */
  --gold: #d9b779;             /* золото: мелкие детали, номера */
}
```

Шрифты — там же, строками ниже (`--font-display`, `--font-body`, `--font-mono`).
Сейчас это **Unbounded** (заголовки), **Manrope** (текст) и **JetBrains Mono**
(подписи и номера) — все с поддержкой кириллицы, подключаются в `<head>` файла `index.html`.

> [!TIP]
> Главный акцент — cyan (`--cyan`). Фиолетовый — для свечения и глубины, золото —
> только для мелких деталей. Красный намеренно не используется.

## 🔧 Технические детали

<details>
<summary><b>Видео: бесшовный hero-луп и «оживающие» ролики</b></summary>
<br>

- **Hero** — качественное видео 1920×1080. Две наложенные копии делают crossfade
  на стыке (0.7 с), поэтому ролик крутится непрерывно и **никогда не «обрывается»** в конце.
- **Шесть игровых роликов** (480p) показываются только внутри рамок галереи: на
  десктопе играют **по наведению мыши**, на тач-устройствах — когда блок виден.
- **Одно видео на всей странице** в любой момент — hero, один игровой ролик или
  видео в окне демо; остальные ставятся на паузу.
- Постер показывается мгновенно, видео появляется плавным crossfade; при скрытой
  вкладке всё встаёт на паузу.
</details>

<details>
<summary><b>3D-сцена и скролл</b></summary>
<br>

- Декоративная сцена на Three.js: звёздное поле, тонкие орбитальные кольца и
  вращающийся кристалл — атмосфера, а не тяжёлое 3D. Один `<canvas>` на фоне, ниже всего контента.
- Плавный скролл — Lenis, приводится одним циклом через `gsap.ticker` (нет борьбы
  за позицию скролла). Навигация по меню мягко доезжает до секций.
- Появления: clip-path у медиа, stagger у текста, лёгкий parallax внутри рамок,
  3D-tilt по наведению, мерцание заголовка манифеста.
- Ambient-звук (оригинальный, сгенерирован кодом) включается **только по кнопке «Звук»**
  с плавным нарастанием — никакого автозапуска.
</details>

<details>
<summary><b>Производительность и доступность</b></summary>
<br>

- Лёгкий режим на мобильных: меньше частиц, приглушённая сцена, видео по видимости.
- При слабом FPS сцена автоматически отключается — контент остаётся полным.
- Без WebGL — просто нет декоративной сцены, сайт работает.
- `prefers-reduced-motion` и `Save-Data` — статичные постеры, без видео и тяжёлой анимации.
- Семантическая разметка, один `H1`, видимый фокус, окно демо на `<dialog>` с
  закрытием по Escape и возвратом фокуса, `aria-pressed` на фильтрах.
- SRI-хэши на CDN-скриптах, ноль ошибок в консоли, ноль горизонтальной прокрутки.
</details>

## ❓ Если что-то пошло не так

| Проблема | Решение |
|---|---|
| Чёрный или пустой экран | Вы открыли файл двойным кликом (`file://`). Запустите локальный сервер — см. [«Как запустить»](#-как-запустить) |
| Не грузятся шрифты / 3D | Проверьте интернет — библиотеки и шрифты идут с CDN |
| Видео не играет / стоит постер | Так и задумано при «уменьшении движения» или экономии трафика (Save-Data). На десктопе игровые ролики играют **по наведению мыши** |
| Тормозит на слабом устройстве | Это учтено: шаблон сам включит облегчённый режим и при нужде отключит 3D-сцену |
| Анимации не двигаются, скролл обычный | В системе включено «уменьшение движения» (reduced motion) — шаблон честно его уважает |
