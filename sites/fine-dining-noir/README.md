## 🚀 Как запустить

Понадобится любой локальный сервер — двойной клик по `index.html` **не сработает**
(браузеры блокируют 3D-модули у файлов, открытых «с диска», адрес `file://`).

**Вариант 1 — терминал (самый простой):**

```bash
cd sites/fine-dining-noir
npx serve .
```

Откройте адрес, который появится в терминале (обычно `http://localhost:3000`).

**Вариант 2 — VS Code / WebStorm:**
правый клик по `index.html` → «Open with Live Server» (в WebStorm — открыть через встроенный браузерный сервер).

**Вариант 3 — Python (если есть):**

```bash
cd sites/fine-dining-noir
python3 -m http.server 8000
# открыть http://localhost:8000
```

> [!IMPORTANT]
> Нужен интернет: Three.js, GSAP, Lenis и шрифты подключаются через CDN.

## 🎨 Тема и палитра

Настроение — темно, тепло, интимно: свет свечей и золота, вкус в центре внимания.

| Цвет | HEX | Где используется |
|---|---|---|
| Почти чёрный | `#0E0B08` | фон, темнота вокруг пятна света |
| Винный | `#4A1220` | панели, акценты, свечение |
| Золото | `#C7962E` | кнопки, лейблы, ключевой свет |
| Кремовый | `#EDE4D3` | текст |

Оба с поддержкой кириллицы, подключаются в `<head>` файла `index.html`.

## ✏️ Как сделать сайт своим

Всё редактируется в обычном текстовом редакторе. Основной файл — `index.html`,
цвета и шрифты — в начале `css/style.css`. Идите по шагам:

### Шаг 1. Название ресторана

Откройте `index.html` и через поиск (`Ctrl+F` / `Cmd+F`) найдите все `NOCTURNE`.
Название встречается в четырёх местах:

| Где | Что искать |
|---|---|
| Вкладка браузера | `<title>` в самом верху файла |
| Экран загрузки | блок `id="preloader"` |
| Шапка сайта | ссылка `class="brand"` в `<header>` |
| Подвал | ещё одна ссылка `class="brand"` внизу + строка копирайта |

### Шаг 2. Тексты, меню и сеты

Каждая секция в `index.html` подписана своим `id` — просто найдите нужную и поменяйте текст внутри:

| Секция | `id` в файле | Что внутри |
|---|---|---|
| Первый экран | `id="hero"` | Заголовок, подзаголовок, кнопки |
| Философия | `id="philosophy"` | Три принципа кухни и манифест |
| Меню / сеты | `id="menu"` | Дегустационное меню (подачи I–VI) и сеты |
| Шеф и команда | `id="chef"` | Цитата шефа, цифры, отзывы гостей |
| Атмосфера | `id="atmosphere"` | Описание зала |
| Бронирование | `id="reserve"` | Форма и контакты |

Все места с выдуманным контентом помечены словами **«ДЕМО»** и **«ЗАГЛУШКА»** —
пройдитесь поиском по ним, чтобы ничего не пропустить.

### Шаг 3. Контакты

Телефон, почта и часы работы — в двух местах: блок `class="reserve__contacts"`
(рядом с формой) и подвал `<footer>`.

### Шаг 4. Форма брони

Сейчас форма — демонстрационная: при отправке показывает «успех», но **данные никуда не уходят**
(обработчик — в `js/main.js`, блок с комментарием «демо-форма брони»). Чтобы брони реально приходили,
подключите любой сервис форм ([Formspree](https://formspree.io/), [Web3Forms](https://web3forms.com/) и т.п.)
или свой бэкенд: добавьте форме `action` или замените обработчик `submit` на отправку `fetch`-ом.

### Шаг 5. Цвета и шрифты (по желанию)

Вся палитра собрана в самом начале `css/style.css`, в блоке `:root`:

```css
:root {
  --c-black: #0E0B08;  /* почти чёрный фон */
  --c-wine:  #4A1220;  /* винный: панели, свечение */
  --c-gold:  #C7962E;  /* золотой акцент: кнопки, детали */
  --c-cream: #EDE4D3;  /* цвет текста */
}
```

Шрифты — там же (`--font-display`, `--font-body`).

> [!TIP]
> Цвета 3D-сцены (свет свечи, вино, окружение) задаются отдельно — в `js/scene.js`.
> Если меняете палитру кардинально, загляните и туда.

### Шаг 6. Своё 3D-блюдо или HDRI (по желанию)

Сейчас вся графика рисуется кодом — внешних файлов нет вообще. Если хотите вставить
настоящую 3D-модель блюда (GLB) или панораму зала (HDRI) — см. короткий гайд в
[`assets/README.md`](assets/README.md).

## 📁 Что в какой папке

```
index.html      вся разметка страницы + подключение библиотек с CDN
css/style.css   все стили: палитра, адаптив, фолбэки
js/main.js      «дирижёр»: прелоадер, запуск сцены и скролла, форма, очистка
js/scene.js     3D-сцена: стол, тарелка, бокал, капля, пар, свеча, постобработка
js/scroll.js    скролл-анимации: маршрут камеры, появление контента, счётчики
assets/         пусто — вся графика процедурная; сюда кладите свои модели/текстуры
```

## 🔧 Технические детали

<details>
<summary><b>3D-сцена и постобработка</b></summary>
<br>

- Процедурное тёплое окружение (шейдерный купол) + PMREM из него же —
  HDRI-подобные отражения на тарелке, бокале и капле без единого внешнего файла.
- Натюрморт из примитивов: тарелка (lathe-профиль), минималистичная подача,
  бокал вина, застывшая капля, свеча с мерцающим пламенем.
- Пар — восходящие additive-спрайты; частицы в луче — GPU-анимация в шейдере;
  дальние боке-огни зала проявляются на отъезде камеры.
- Постобработка: UnrealBloom (свет свечей), BokehPass (глубина резкости, наводка на блюдо),
  кастомная цветокоррекция под палитру — винные тени, золотые света, виньетка, зерно.
- Один shadow-casting spotlight создаёт тёплый пул света и глубокие тени.
</details>

<details>
<summary><b>Кинематография скролла</b></summary>
<br>

- Lenis (плавный скролл) + GSAP ScrollTrigger.
- Мастер-таймлайн камеры с мягким easing (power2/power3) и паузами между секциями.
- Вступление: камера наводит фокус из расфокуса на каплю и блюдо (rack focus, ~5 с).
- Маршрут: герой → дрейф к свече → взгляд сверху на подачу → бокал → отъезд с боке зала → наезд к свече.
- Появление контента со stagger, счётчики цифр, плавные якоря.
</details>

<details>
<summary><b>Производительность и доступность</b></summary>
<br>

- Лёгкий режим 3D на мобильных и слабых устройствах: меньше частиц, без depth of field
  и теней, упрощённые материалы, сниженный pixel ratio + автоматическая деградация при падении FPS.
- Без WebGL — фолбэк на CSS-градиент, сайт остаётся читаемым.
- `prefers-reduced-motion` — статичный кадр, без Lenis и полёта камеры.
- Полная очистка ресурсов при уходе со страницы: dispose геометрий/материалов/текстур,
  kill ScrollTrigger, destroy Lenis.
- SRI-хэши на CDN-скриптах, ноль ошибок в консоли.
</details>

## ❓ Если что-то пошло не так

| Проблема | Решение |
|---|---|
| Чёрный или пустой экран | Вы открыли файл двойным кликом (`file://`). Запустите локальный сервер — см. [«Как запустить»](#-как-запустить) |
| Нет шрифтов / 3D не грузится | Проверьте интернет — библиотеки и шрифты идут с CDN |
| Тормозит на слабом устройстве | Это учтено: шаблон сам включит облегчённый режим. Если всё равно тяжело — уменьшите число частиц в `js/scene.js` |
| Анимации не двигаются | Возможно, в системе включено «уменьшение движения» (reduced motion) — шаблон честно его уважает |
