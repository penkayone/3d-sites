Я добавил новый 3D-шаблон в этот репозиторий. Оформи его для GitHub так же, как оформлены остальные шаблоны. Путь к шаблону: $ARGUMENTS (если путь не указан — определи новый шаблон сам по `git status` или по последнему коммиту).

## Стиль (главное)

- Всё на русском, **простыми человеческими словами**, без жаргона. Читатель — человек, который просто копирует шаблон и не должен испытывать трудностей.
- Оформление профессиональное: бейджи, таблицы, центрированная шапка, блоки `> [!IMPORTANT]` / `> [!TIP]` / `> [!NOTE]`.
- Технические детали не удалять, а прятать в сворачиваемые блоки `<details>`.
- **Эталон структуры и тона — `aviation/jet-charter-dawn/README.md`.** Сначала прочитай его и корневой `README.md`, потом пиши.

## Шаг 1. Скриншот первого экрана

1. Запусти локальный сервер в папке шаблона в фоне: `python3 -m http.server 8471`.
2. Сними кадр (ожидание нужно, чтобы успели пройти прелоадер и интро-анимация):
   ```bash
   npx -y playwright screenshot --channel chrome --viewport-size "1600,900" \
     --wait-for-timeout 15000 "http://localhost:8471/index.html" preview.png
   ```
3. **Посмотри на картинку глазами** (Read). Если в кадре прелоадер или пустой экран — увеличь ожидание и повтори.
4. Сожми и положи на место, PNG удали, сервер останови:
   ```bash
   sips -s format jpeg -s formatOptions 82 preview.png --out docs/previews/<имя-шаблона>.jpg
   ```

## Шаг 2. README шаблона (пошаговый гайд)

Структура — как в эталоне:

1. Центрированная шапка: название, одно предложение о «сюжете» сайта, скриншот (`../../docs/previews/<имя>.jpg`, width 720).
2. `[!NOTE]`: это шаблон, бренд/цифры/отзывы — демо-заглушки.
3. **«Как запустить»** — три варианта (npx serve, Live Server, python3 -m http.server), предупреждение, что двойной клик по index.html не сработает (`file://`) и что нужен интернет (CDN).
4. **«Как сделать сайт своим»** по шагам:
   - название бренда — найди в коде ВСЕ места, где оно встречается, и перечисли таблицей;
   - таблица секций: `id` → что внутри (возьми реальные id из index.html);
   - контакты — где лежат;
   - форма — проверь в js, отправляет ли она данные; если это демо-заглушка, честно напиши и посоветуй Formspree/Web3Forms или свой бэкенд;
   - цвета и шрифты — покажи реальный блок `:root` из css с комментариями;
   - свои ассеты — ссылка на assets/README.md, если есть.
5. **«Что в какой папке»** — короткая таблица файлов.
6. Техдетали (сцена, скролл, производительность/доступность) — в `<details>`.
7. Таблица **«Если что-то пошло не так»**: чёрный экран (file://), нет интернета, тормозит, reduced motion.

Все id, классы, переменные и имена функций бери из реального кода шаблона — ничего не выдумывай.

## Шаг 3. assets/README.md (если папка есть)

По образцу эталонного: зачем папка, как подключить свою GLB-модель / HDRI / текстуры — с рабочими сниппетами кода и ссылками (Sketchfab, Poly Haven, gltf.report).

## Шаг 4. Корневой README.md

Добавь строку в таблицу «Шаблоны»: превью `width="420"` со ссылкой на папку, название со ссылкой на README шаблона, описание в 1–2 предложения, статус ✅. Если ниша была в списке «запланированных» — убери её оттуда. Проверь, что категория есть в таблице категорий.

## Шаг 5. Git

- Ветка `feature/<имя-шаблона>` (или `feature/docs-<имя-шаблона>`, если код уже в main).
- Если код шаблона ещё не закоммичен — сначала отдельный коммит `feat(<категория>): add <имя-шаблона> 3d landing template`.
- Документация и превью — отдельным коммитом `docs(<категория>): add <имя-шаблона> readme and preview`.
- Пуш, PR через `gh pr create` (описание PR на русском, по пунктам), затем `gh pr merge --squash --delete-branch`, обнови локальный main.

## Ограничения

- Код самого шаблона не менять (кроме случая, когда он ещё не закоммичен — тогда просто закоммитить как есть).
- Никаких новых зависимостей и сборщиков.
- В конце покажи ссылку на PR и что именно изменилось.
